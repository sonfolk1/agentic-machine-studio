# Agentic Studio Mobile

iPhone companion to the Agentic Studio desktop app. Pairs over LAN
WebSocket (`ws://<computer-ip>:8765`), runs the OpenRouter agent loop
on the phone, and forwards tool calls (`shell`, `create_file`, …) to
the desktop for execution.

## Quickest path to a sideloadable .ipa

> Requires Xcode installed (free, ~12 GB from the Mac App Store).
> The Command Line Tools alone aren't enough — they don't ship the iOS SDK.

```bash
# 1) Once Xcode is installed, point xcode-select at it:
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer

# 2) Build the unsigned .ipa:
cd ~/Downloads/agentic-studio-mobile
./build.sh

# 3) Output:
ls -lh build/AgenticStudioMobile.ipa
```

Then on your phone:

1. AirDrop or iCloud-share `build/AgenticStudioMobile.ipa` to the iPhone.
2. Open **SideStore → My Apps → + → pick the .ipa**.
3. SideStore re-signs with your Apple ID and installs. The app appears
   on your home screen.
4. Refresh in SideStore once a week so the free-Apple-ID signature
   stays valid.

## Regenerating the project file

Add or remove Swift files freely. To re-sync the Xcode project:

```bash
node scripts/genproj.js
```

`build.sh` runs the generator automatically on every build.

## Pairing with the desktop

1. On the desktop running Agentic Studio, find the LAN IP:

   ```bash
   ipconfig getifaddr en0
   ```

2. Launch the desktop app. The first time it binds `0.0.0.0:8765`,
   macOS will prompt the firewall — click **Allow**.

3. On the iPhone, open **Agentic Studio**, type `ws://<that-ip>:8765`
   into the dashboard's connection box, tap **Connect**. The status
   capsule turns 🟢 "Remote Active".

4. Open the gear → Chat Settings → paste your OpenRouter API key
   (stored in the iOS Keychain only).

5. Type a task; the dashboard slides up, the conversation slides in,
   tool calls fan out to the desktop.

## Wire protocol

Mobile → desktop:

```json
{"id":"…","type":"shell","command":"ls -la"}
{"id":"…","type":"create_file","path":"hello.txt","contents":"hi"}
{"id":"…","type":"edit_file","path":"x.ts","old_string":"foo","new_string":"bar"}
{"id":"…","type":"view_file","path":"src/main.ts"}
{"id":"…","type":"scan_directory","path":".","depth":3}
{"id":"…","type":"workspace_info"}
{"id":"…","type":"ping"}
```

Desktop → mobile: `{id, result}` on success, `{id, error}` on failure.

## Security note

The desktop bridge binds `0.0.0.0:8765` so the iPhone on your LAN can
reach it. Anyone on the same network who knows the port can issue
shell commands. Run on trusted networks only. A future iteration
should add a shared-secret handshake.
